New `swipeToCloseDirection` input (`'up' | 'down'`, default `'up'`) to flip the mobile close-swipe axis
`slideTransition` and `flipTransition` are now re-exported from `@reelkit/angular-lightbox` alongside `lightboxFadeTransition` and `lightboxZoomTransition` — import every built-in transition from a single entry
BREAKING: `transition` string input (`'slide' \| 'fade' \| 'flip' \| 'zoom-in'`) and `TransitionType` export removed. Pass a `TransitionTransformFn` directly to the `transitionFn` input (defaults to `slideTransition`). Unused built-ins now tree-shake out (~2 kB gzip saving when you only use one). Migrate `transition="fade"` → `[transitionFn]="fadeTransition"` after binding a class field to the imported `lightboxFadeTransition` from `@reelkit/angular-lightbox`.
