New transition engine — transitions are now pure functions (TransitionTransformFn) that receive slide progress and return CSS transforms, replacing the previous string-based system. Build fully custom transitions or use built-in ones (slide, fade, flip, cube, zoom) as tree-shakeable imports. getSlideProgress helper for computing normalized slide offsets in custom transitions. Tap, double-tap, and long-press gesture detection. Sound controller, content loading controller, content preloader with LRU cache. Fullscreen utilities — fullscreenSignal (lazy singleton), requestFullscreen and exitFullscreen with Safari vendor-prefix guards.
